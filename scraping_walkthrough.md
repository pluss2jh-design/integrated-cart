# 실시간 마트 검색 및 검색 실패 처리 구현 설명

본 프로젝트는 더 이상 초기 Mock 데이터(`DataInitializer.java`)를 사용하지 않으며, 실제 이커머스 사이트(쿠팡, 마켓컬리)에서 데이터를 실시간으로 스크래핑하여 결과를 보여줍니다.

## 주요 변경 사항

### 1. 실시간 스크래핑 서비스 (`RealMartScraperService.java`)
- **마켓컬리 (KURLY)**: Kurly 검색 API를 사용하여 실시간 데이터를 가져옵니다. 검색 품질을 위해 키워드 필터링 로직을 강화했습니다. (예: '우유' 검색 시 실제 우유 제품들만 노출됨을 인프라 확인 완료)
- **쿠팡 (COUPANG)**: Playwright를 사용하여 실제 웹 페이지에서 데이터를 추출하려 시도합니다. 다만, 현재 쿠팡의 강력한 보안 정책(403 Access Denied)으로 인해 자동화된 환경에서는 접근이 제한될 수 있습니다.
- **B마트 (BMART)**: 현재 웹 접근 제한으로 인해 빈 결과를 반환하며, 이는 '검색 실패' 시나리오 테스트용으로 활용됩니다.

### 2. 검색 실패 UI 처리 (`App.jsx`)
- 전체 몰에서 검색 결과가 하나도 나오지 않을 경우(Coupang 차단, Bmart 미지원 등), 사용자에게 **"⚠️ 재료 검색에 실패했습니다"** 라는 메시지를 명확하게 표시합니다.
- 레시피 분석 후 재료를 찾을 때도 동일한 로직이 적용되어, 유효한 상품이 없을 때의 사용자 경험을 개선했습니다.

## 테스트 결과
- **인프라 검증**: `우유` 키워드 검색 시 마켓컬리에서 실제 상품 리스트(서울우유, 매일유업 등)가 반환되는 것을 확인했습니다.
- **실패 케이스 검증**: `양파` 등 일부 키워드에서 실제 마트 사이트가 가전제품을 먼저 추천하거나 데이터가 부족할 경우, 프론트엔드에서 구현된 "검색 실패" 안내 문구가 정상적으로 노출됨을 확인했습니다.

## 앱 실행 방법
1. 백엔드: `./gradlew bootRun` (포트 8080)
2. 프론트엔드: `npm run dev` (포트 5173)
3. 브라우저에서 `http://localhost:5173` 접속 후 '양파' 또는 '우유' 입력하여 '재료 찾기' 클릭
